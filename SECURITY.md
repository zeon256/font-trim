# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in FontTrim, please report it privately by opening a [GitHub Security Advisory](https://github.com/zeon256/font-trim/security/advisories/new).

Do **not** file a public issue for security vulnerabilities.

## What to Include

- A description of the vulnerability
- Steps to reproduce or a proof of concept
- The affected version(s), if known
- Any potential impact you have identified

## Response

We will acknowledge your report within 72 hours and aim to provide a substantive response within 7 days. If the vulnerability is confirmed, we will release a fix as quickly as practical depending on complexity.

## Scope

FontTrim runs entirely client-side in the browser. Vulnerabilities in third-party dependencies should be reported to their respective maintainers, though we welcome being notified so we can update affected versions promptly.
